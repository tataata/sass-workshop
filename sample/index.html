Preprocessors for CSS. What for?

back to top

CSS preprocessors good for using because they give to us a possibility to use variables and reuse them through whole file. Variables, guys! We all dreamed about it!

When we write CSS code we have to remember different values and relations between selectors, describe homogeneous elements with small differences and in the end we have a long poem with a lot of repetitions.

CSS preprocessors take code written in the preprocessed language and then convert that code into the same CSS we’ve been writing for years.

Three of the most popular CSS preprocessors are SASS, LESS, and Stylus. In general all of them the same. More detail information about differences between them you can rich in Johnathan Croom’s section “Sass vs. LESS vs. Stylus: Preprocessor Shootout” (http://net.tutsplus.com/tutorials/html-css-techniques/sass-vs-less-vs-stylus-a-preprocessor-shootout/).

Here we will do an overview of SASS.

Introduction to SASS

back to top

SASS is an acronym for a Syntactically Awesome Stylesheets. Official about SASS located on http://sass-lang.com/. Documentation available here: http://sass-lang.com/docs.html

There are two syntaxis in SASS:

  1. SCSS, that is the new main syntax. It is very similar with CSS syntax with its curly braces. Every valid CSS3 stylesheet is valid SCSS as well. SCSS files use the extension .scss.

    $blue: #3bbfce; $margin: 16px; .content-navigation { border-color: $blue; color: darken($blue, 9%); } .border { padding: $margin / 2; margin: $margin / 2; border-color: $blue; }

    In this workshop we will use this syntax.

  2. SASS, the old one version of syntax, aka the indented syntax. Instead of brackets and semicolons, it uses the indentation of lines to specify blocks. As authors say, the indented syntax will continue to be supported. Files in the indented syntax use the extension .sass. More about it you can read here http://sass-lang.com/docs/yardoc/file.INDENTED_SYNTAX.html

    $blue: #3bbfce

    $margin: 16px

    .content-navigation

      border-color: $blue

      color: darken($blue, 9%)

    .border

      padding: $margin / 2

      margin: $margin / 2

      border-color: $blue

Let’s see what else SASS allows to use.

SASS: inheritance, nesting, variables, mixins

back to top

For saving ourselves from a lot of duplicates in SAS we can use nesting and selector inheritance.

Inheritance

When we want to implement the same styles to multiple elements, in CSS we use the following construction:

p,

ul,

ol {

  /* styles here */

}

That works great, but if we need to further style the elements individually, another selector has to be created for each and it can quickly get messier and harder to maintain. To counter this, inheritance can be used. Inheritance is the ability for other CSS selectors to inherit the properties of another selector.

For avoiding of duplicate properties in SASS we can to extend them via @extend.

.error {

  border: 1px #f00;

  background: #fdd;

}

.error.intrusion {

  font-size: 1.3em;

  font-weight: bold;

}

.badError {

  @extend .error;

  border-width: 3px;

}

Nesting

For describing properties of child elements with SASS we can to nest it in curly brackets of parent element.

section {

  margin: 10px;

  nav {

    height: 25px;

    li {

      float: left;

    }

  }

}

Variables

Variables can be declared and used throughout the stylesheet. They can have any value that is a CSS value (e.g. colors, numbers, or text.), and can be referenced anywhere throughout stylesheet.

Before the name of variable need to use the $ symbol and the value and name are separated with a semicolon, like a CSS property.

$mainColor: #0982c1;

$siteWidth: 1024px;

mainColor: #0982c1;

$borderStyle: dotted;

body {

  color: $mainColor;

  border: 1px $borderStyle $mainColor;

  max-width: $siteWidth;

}

Mixins

Mixins indent for more complicated staff than variables. Mixins allow you to re-use whole fragments of CSS, properties or selectors. You can even give them arguments.

Installation

For running SASS on your computer you need a Ruby first.

Mac

For check if you have Ruby or not type in Terminal ruby -v

On Mac OS X and older it is preinstalled. If not then use the guide for installiation on http://net.tutsplus.com/tutorials/ruby/how-to-install-ruby-on-a-mac/

Windows

Windows users can download Ruby from http://rubyinstaller.org/downloads/ or http://rubyinstaller.org/. This is a self-contained Windows-based installer.

Linux

Linux users can install it via their package manager.

Ok, let’s install Sass! Open up your Terminal.app and type:

OS X / Linux

$ sudo gem install sass

Windows

$ gem install sass

Installation from Git

Sass can also be installed directly from the Git repository:

clone git://github.com/nex3/sass.git

cd sass

rake install

Importing

In the CSS community, importing CSS is frowned upon because it requires multiple HTTP requests. Importing with a preprocessor works differently, however. If you import a file from any of the three preprocessors, it will literally include the import during the compile, creating only one file. Keep in mind though that importing regular .css files compiles with the default @import "file.css"; code. Also, mixins and variables can be imported and used in your main stylesheet. Importation makes creating separate files for organization very worthwhile.
